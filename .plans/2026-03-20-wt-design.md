# `wt` — Worktree Manager

## Motivation

Working with AI coding agents across multiple branches means repetitive setup for each task: create a git worktree, open a tmux window, copy config files, launch the agent. Reverse it all when you're done. Multiply by several concurrent tasks and it's a lot of ceremony.

`wt` reduces that to `wt add <branch>` and `wt rm <branch>`. One command to spin up a fully configured workspace — worktree, tmux window, config files, agent launch. One command to tear it down. One command (`wt attach`) to jump between them. It's agent-agnostic — configure it to launch Claude Code, Cursor, or anything else.

## Commands

| Command | Purpose |
|---------|---------|
| `wt add <branch> [-a]` | Create git worktree + tmux window. Run `launch_command` if configured. `-a` attaches after creation. |
| `wt rm <branch> [-d\|-D]` | Remove worktree + tmux window. `-d` deletes branch, `-D` force-deletes. |
| `wt attach [branch]` | Attach to tmux session. If branch given, switch to that window. |
| `wt config path` | Print config file path. |
| `wt config edit` | Open config in `$EDITOR`. Creates default if missing. |
| `wt config refresh` | Create or update config file with default values. |
| `wt completion [bash\|zsh\|fish]` | Generate shell completions. |
| `wt --verbose` | Global flag for debug output. |

All commands except `attach` must be run from the main repository (not a worktree).

## Paths & Naming

| Resource | Path |
|----------|------|
| Worktree base | `~/.local/share/wt/worktrees/<repo>/` |
| Worktree | `~/.local/share/wt/worktrees/<repo>/<repo>-<sanitized-branch>` |
| Config file | `~/.config/wt/config.json` |
| Tmux socket | `-L wt` |
| Tmux session | `wt-<reponame>` |
| Tmux window | `<sanitized-branch>` |

Branch sanitization: replace `[^a-zA-Z0-9-]` with `-`.

All paths follow XDG conventions (`XDG_DATA_HOME`, `XDG_CONFIG_HOME`).

## Configuration

```json
{
  "launch_command": "",
  "setup_scripts": [],
  "copy_files": []
}
```

- Auto-created with defaults on first run of any command that needs it.
- **`launch_command`** — command sent to tmux window after worktree creation. Empty = no command.
  - Example: `"claude --permission-mode acceptEdits"`
- **`setup_scripts`** — list of scripts to run in the new worktree after creation. Paths are relative to the worktree root. Scripts that don't exist are silently skipped. All matching scripts are run in order.
  - Example: `["scripts/setup.sh", "scripts/install-deps.sh"]`
- **`copy_files`** — list of files to copy from the main repo to the new worktree. Paths are relative to the repo root. Files that don't exist are silently skipped.
  - Example: `[".claude/settings.local.json", ".env.local"]`

## `wt add` Flow

1. Validate: must be run from main repo (not a worktree).
2. Ensure tmux session `wt-<reponame>` exists (create if not).
3. Create git worktree at `~/.local/share/wt/worktrees/<repo>/<repo>-<branch>`. Skip if already exists.
4. Copy `copy_files` from main repo to worktree. Paths relative to repo root. Skip files that don't exist.
5. Run `setup_scripts` in the worktree. Paths relative to worktree root. Skip scripts that don't exist.
6. Create tmux window `<sanitized-branch>` in the session, cd'd to the worktree. Skip if already exists.
7. If `launch_command` is non-empty, send it to the tmux window via `tmux send-keys`.
8. If `-a` flag is set, attach to the session/window.

## `wt rm` Flow

1. Remove git worktree via `git worktree remove`.
2. Kill tmux window for that branch.
3. If `-d` or `-D`, delete the git branch.

## `wt attach` Flow

1. No args: attach to the `wt-<reponame>` tmux session.
2. With branch arg: attach and switch to that branch's window.
3. If already inside the `wt` tmux socket, use `switch-client` instead of `attach-session`.

## Architecture

### Go Module

```
agent-tools/              # repo root
├── go.work               # Go workspace: references worktree-manager/
├── .gitignore
├── LICENSE
├── README.md
└── worktree-manager/
    ├── go.mod             # module github.com/averycrespi/agent-tools/worktree-manager
    ├── main.go            # Entry point: cmd.Execute()
    ├── cmd/
    │   ├── root.go        # Root command, --verbose flag, service construction, completion
    │   ├── add.go
    │   ├── rm.go
    │   ├── attach.go
    │   └── config.go
    └── internal/
        ├── workspace/     # Core orchestration: Add, Remove, Attach
        ├── git/           # Git operations: AddWorktree, RemoveWorktree, RepoInfo, etc.
        ├── tmux/          # Tmux operations: CreateSession, CreateWindow, SendKeys, etc.
        ├── config/        # JSON config: Load, Save, defaults, XDG paths, branch sanitization
        └── exec/          # Command runner interface (for testability)
```

### Dependency Injection

Services accept interfaces for git and tmux (for testability via `exec.Runner`). Config is passed as a plain struct value — loaded once at startup in `root.go`'s `PersistentPreRun`, which also constructs the services.

```go
type Service struct {
    git    gitClient
    tmux   tmuxClient
    config config.Config
    logger *slog.Logger
    runner exec.Runner
}
```

Logging uses `log/slog` from the standard library. Verbose flag controls log level (default `Warn`, verbose `Debug`).

### Error Handling

- All Cobra commands use `RunE` (not `Run`) to return errors.
- Root command sets `SilenceUsage: true` and `SilenceErrors: true` — runtime errors don't dump usage text.
- `main.go` handles the top-level error: log it with `slog.Error` and `os.Exit(1)`.
- Wrap errors with context using `%w`: `fmt.Errorf("failed to create worktree %q: %w", name, err)`.

### Testing

- Unit tests use mock implementations of interfaces (testify/mock).
- Integration tests require tmux.
- Test naming: `TestType_Method_Scenario`.
- Always run with race detection: `go test -race ./...`.

### Linter Configuration

`.golangci.yml` (v2 format) at the `worktree-manager/` root:

- `default: standard` enables the standard linter set (govet, staticcheck, errcheck, unused, ineffassign).
- Additional linters: `errorlint`, `gocritic`, `gosec`.
- Formatters: `goimports`.
- Suppress `gosec` in test files.

### Makefile

A `Makefile` at the `worktree-manager/` root with these targets:

| Target | Command |
|--------|---------|
| `build` | `go build -o /tmp/bin/wt .` |
| `test` | `go test -race ./...` |
| `lint` | `go tool golangci-lint run` |
| `fmt` | `go tool goimports -w .` |
| `tidy` | `go mod tidy && go mod verify` |
| `audit` | `tidy` + `fmt` + `lint` + `test` + `go tool govulncheck ./...` |

`make audit` is the local equivalent of CI — run it before pushing.

External tools shelled out to: `git`, `tmux`.
